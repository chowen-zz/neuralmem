## 二、遗忘曲线衰减（DecayManager）

### 算法

Ebbinghaus 指数衰减公式：

```
R = e^(-t / S)
```

- `R`：记忆保留率（0.0–1.0）
- `t`：距上次访问的天数（`datetime.now() - last_accessed`）
- `S`：记忆稳定性，初始值 = `memory.importance × 10`，每次访问后 `S *= 1.5`

**稳定性说明：**  
高频访问的记忆稳定性增长，衰减更慢；长期未访问的记忆 R 迅速趋近于 0。

### 数据来源

直接读取 SQLite `memories` 表已有字段，**无需新字段或迁移**：

| 字段 | 用途 |
|------|------|
| `importance` | 初始稳定性种子，衰减后更新此字段 |
| `last_accessed` | 计算 t（天数） |
| `access_count` | 累积访问次数，用于计算实际稳定性 `S = importance × 10 × 1.5^access_count` |

### apply_decay(user_id) 行为

1. 查询指定 user_id 的所有记忆（或全部，若 user_id=None）
2. 计算每条记忆当前 R 值
3. 将 `importance` 更新为 `old_importance × R`（衰减后的重要性）
4. 返回更新条数

### remove_forgotten(user_id) 行为

1. 物理删除 `importance < 0.05` 的记忆
2. 同步清理 `memories_fts` 和 `memories_vec` 中的对应记录
3. 返回删除条数

### 触发方式

仅由用户显式调用 `mem.consolidate()` 触发，**不自动运行**。

## 三、Cross-Encoder 重排序（CrossEncoderReranker）

### 模型

`cross-encoder/ms-marco-MiniLM-L-6-v2`

- 大小：~67MB
- 速度：CPU 上 < 50ms/10条（适合本地使用）
- 效果：英文检索重排序基准最佳小模型之一
- 安装：`pip install neuralmem[reranker]`

### 懒加载 + 优雅降级

```python
def _get_model(self):
    if self._model is None:
        try:
            from sentence_transformers import CrossEncoder
            self._model = CrossEncoder("cross-encoder/ms-marco-MiniLM-L-6-v2")
        except ImportError:
            _logger.info(
                "sentence-transformers not installed. "
                "Run 'pip install neuralmem[reranker]' to enable cross-encoder reranking."
            )
            self._model = False  # False = 已检测，不可用（区别于 None = 未检测）
    return self._model or None
```

### 接口签名修正

当前 stub 签名 `rerank(query, list[tuple[str, float]])` 只有 ID，无法获取内容。
**实际实现需要 Memory 内容**，因此签名升级为：

```python
def rerank(
    self,
    query: str,
    candidates: list[tuple[Memory, float]],  # Memory 对象而非 ID
) -> list[tuple[str, float]]:                 # 返回 (id, new_score)
```

### rerank() 逻辑

1. `model = None` → 原顺序返回 `[(m.id, score) for m, score in candidates]`（降级）
2. `len(candidates) <= 1` → 跳过计算，直接返回
3. 批量计算 `CrossEncoder.predict([(query, m.content) for m, _ in candidates])` 得分
4. 按新得分降序排列并返回 `list[tuple[str, float]]`

### 引擎侧配套修改（engine.py）

`RetrievalEngine.search()` 中在调用 rerank 前先加载记忆对象（约 3 行改动）：

```python
# 原来：直接传 (id, score) 对
# 修改后：先加载 Memory 对象再传入
if self._config.enable_reranker:
    loaded = [(self._storage.get_memory(mid), s) for mid, s in top_candidates if self._storage.get_memory(mid)]
    top_candidates_ids = self._reranker.rerank(query.query, loaded)
    top_candidates = top_candidates_ids
```

### 接入点

用户通过配置启用（`engine.py` 改动约 5 行，接口稳定）：

```python
mem = NeuralMem(config=NeuralMemConfig(enable_reranker=True))
```

## 四、实体消歧（EntityResolver）

### 文件

新增 `src/neuralmem/extraction/entity_resolver.py`，不修改 `extractor.py`。

### 两阶段流程

```
新实体 "Alice"  +  图谱中已有实体列表
         │
         ├─ 阶段1: 规则快速过滤（O(n) 字符串操作，零 ML 延迟）
         │   ├─ 完全匹配（name == existing.name）→ 直接确认合并
         │   ├─ 子串匹配（name ⊂ existing.name 或反向）→ 加入候选
         │   ├─ 编辑距离 ≤ 2 → 加入候选
         │   └─ 无候选 → 直接创建新实体（跳过阶段2）
         │
         └─ 阶段2: Embedding 精排（仅对候选触发）
             └─ cosine(embed(new_name), embed(existing_name)) ≥ 0.85
                 ├─ True  → 确认合并（取已有实体，添加 alias）
                 └─ False → 创建新实体
```

### 合并行为

**不删除原实体**，而是：
1. 将新实体 `name` 追加到已有实体的 `aliases` 列表
2. 更新 `last_seen` 时间戳
3. 返回已有实体（而非新建）

图谱中只有一个节点，新名字作为别名存在，解决"同一人不同称呼"问题。

### 接口

```python
class EntityResolver:
    def resolve(
        self,
        new_entities: list[Entity],
        existing_entities: list[Entity],
    ) -> list[Entity]:
        """
        对每个 new_entity，判断是否与 existing_entities 中的某个匹配。
        匹配则返回合并后的已有实体，不匹配则返回原新实体。
        """
```

### 性能考虑

- 阶段1 规则过滤通常可淘汰 95% 的候选，绝大多数不触发 Embedding
- Embedding 只在有规则候选时才调用，且仅对候选子集计算（通常 1-3 个）
